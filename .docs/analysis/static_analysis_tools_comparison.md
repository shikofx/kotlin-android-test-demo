# Сравнение утилит для статического анализа

В проекте используется три основных инструмента для статического анализа кода: **Checkstyle**, **PMD** и **Android Lint**. Может возникнуть вопрос: зачем нужны все три, если они, казалось бы, делают одно и то же?

На самом деле, эти инструменты не заменяют, а **дополняют** друг друга. Каждый из них специализируется на своем типе проблем, создавая многоуровневую систему контроля качества кода.

## Ключевые различия

| Инструмент | Специализация | Примеры проверок |
| :--- | :--- | :--- |
| **Checkstyle** | **"Корректор стиля"** | Следит за тем, *как* код написан. Обеспечивает единообразие и читаемость. <br> - Неправильные отступы <br> - Длина строк <br> - Порядок импортов <br> - Расстановка скобок |
| **PMD** | **"Инженер по качеству Java"** | Следит за тем, *что* делает ваш Java-код с точки зрения общих практик программирования. Ищет "дурно пахнущий код". <br> - Пустые `catch`-блоки <br> - Неиспользуемые переменные <br> - Слишком сложные методы <br> - Создание объектов в цикле |
| **Android Lint** | **"Инженер по качеству Android"** | Следит за тем, *что* делает ваш код в контексте **платформы Android**. <br> - Проблемы производительности в XML <br> - Использование устаревших Android API <br> - Проблемы с безопасностью и доступностью (accessibility) <br> - Жестко закодированные строки |

## Пример, который показывает разницу

Чтобы лучше понять уникальную роль каждого инструмента, представим, что они анализируют один и тот же код:

*   **Checkstyle** скажет вам, что имя переменной `My_VAR` не соответствует стилю `camelCase`. Его не волнует, что делает эта переменная, только как она названа.

*   **PMD** скажет, что вы создаете новый объект `Paint` внутри цикла `for`, что неэффективно и создает лишнюю нагрузку на сборщик мусора. Он анализирует логику и производительность общего Java-кода.

*   **Android Lint** скажет, что вы используете `android:layout_height="match_parent"` внутри `ScrollView`. Он знает, что это приведет к проблемам с производительностью рендеринга на платформе Android, так как `ScrollView` будет пытаться измерить бесконечную высоту.

## Вывод

Использование всех трех анализаторов — это лучшая практика в индустрии, которая позволяет:
*   **Checkstyle** — поддерживать код чистым и единообразным.
*   **PMD** — избегать распространенных ошибок и плохих практик в Java.
*   **Android Lint** — обеспечивать корректность, производительность и безопасность приложения на платформе Android.

Таким образом, они не дублируют, а работают на разных уровнях, обеспечивая комплексный контроль качества.
